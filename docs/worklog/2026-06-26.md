## Backend

- `shop_candidates` Flyway migration을 추가했다.
- 외부 provider ID 중복 방지를 위해 `(source, source_place_id)` partial unique index를 두었다.
- 검수 상태는 `pending`, `rejected`, `promoted`로 제한하고, `promoted` 상태에는 `promoted_shop_id`가 필요하도록 했다.
- OpenAPI info에서 확정되지 않은 MIT license metadata를 제거했다.
- Docker가 없는 로컬 환경에서 Homebrew PostgreSQL 17/PostGIS로 실제 DB 수동 테스트 환경을 구성했다.
- PostgreSQL nullable parameter 타입 추론 문제로 깨지던 `GET /shops` 목록 조회를 고쳤다.
- 네이버 지역 검색 API 결과를 `shop_candidates`에 upsert하는 `POST /admin/shop-candidates/sync` spike를 추가했다. 검색 결과는 검수 전 후보로만 저장하고 `shops`에는 직접 넣지 않는다.
- Vite dev server에서 Spring API를 호출할 수 있도록 로컬 개발용 CORS 설정을 추가했다.
- V1 초기 스키마에 `shops.thumbnail_url`을 포함하고, 값이 비어 있으면 기본 라멘 placeholder asset 경로를 저장하도록 했다.
