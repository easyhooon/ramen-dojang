## Product / Auth

- 유저별 방문 기록과 위시리스트를 서버에 저장하려면 최소한의 사용자 식별이 필요하므로, 토스 로그인보다 가벼운 앱인토스 `getAnonymousKey` 기반 익명 식별을 우선 검토하기로 정리했다.
- 토스 로그인은 개인정보 조회, 결제 상태 조회, 계정 연결이 필요할 때 별도 작업으로 미룬다.
- 한국 밖 사용자도 고려해 한국어/영어/일본어 다국어 지원을 계획에 추가했다. 기본은 시스템 언어를 따르고, 설정에서 언어를 직접 고정하는 방향이다.

## Deployment

- API/DB 배포 후보별 월비용을 정리했다. 비용만 보면 Render가 가볍지만, 현재는 장기 운영과 자료 접근성을 고려해 AWS 최소 구성을 우선하기로 정리했다.
- AWS 콘솔 진입 이후 현재 스펙에 맞는 최소 setup 문서를 작성했다. API는 Elastic Beanstalk Java 17, DB는 RDS PostgreSQL/PostGIS로 두고, Vercel/앱인토스 origin, seed import, budget, IAM, smoke test 항목을 체크리스트로 정리했다.
