# 2026-07-07 Worklog

## 한 줄 요약

배포 전 개발 단계에 맞춰 `shop_candidates` schema를 V1 초기 migration에 합치고, 로컬 PostgreSQL 17/PostGIS DB를 reset한 뒤 API로 라멘집 데이터가 저장되는 것까지 확인했다.

## Backend

- `V2__create_shop_candidates.sql`을 제거하고 `shop_candidates` 테이블과 관련 index를 `V1__create_initial_schema.sql`에 합쳤다.
- 후보 데이터를 검수하기 위한 admin API 초안을 추가했다.
  - `GET /admin/shop-candidates`
  - `POST /admin/shop-candidates/{candidateId}/promote`
- 후보 승격 시 `shops`에 라멘집을 만들고 `shop_candidates.promoted_shop_id`, `status=promoted`, `reviewed_at`을 갱신하도록 했다.
- 방문 기록이 없는 라멘집의 `averageRating`이 `0.0`으로 내려가던 `ResultSet.wasNull()` 사용 버그를 수정했다.

## DB / Local Infra

- 로컬 DB는 배포 전까지 migration을 계속 쌓기보다 reset 후 V1을 다시 적용하는 흐름으로 정리했다.
- Homebrew `postgresql@16`에는 PostGIS extension 파일이 없어 `postgresql@17` 서비스로 전환했다.
- 앱 계정 `ramen`은 `CREATE EXTENSION postgis` 권한이 없으므로, 로컬 DB 생성 직후 superuser로 PostGIS extension을 먼저 설치하고 서버를 띄웠다.
- reset 후 Flyway가 `Successfully validated 1 migration`, `Successfully applied 1 migration` 상태로 V1만 적용하는 것을 확인했다.
